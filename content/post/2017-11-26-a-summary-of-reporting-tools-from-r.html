Introduction

The data science pipeline often leads to some sort of report, typically a research paper in academia, or a data product in industry. Since this class caters more to the academia/research side, let’s describe some of the ways we can report the results of our data analyses.

Literate programming

The CRAN Task Views page on Reproducible Research describes reproducible research as follows:

The goal of reproducible research is to tie specific instructions to data analysis and experimental data so that scholarship can be recreated, better understood and verified.

The basic idea behind this concept in R is the idea of literate programming, where one creates a document that combines both content and code and uses a program to replace the code with the results of that code so that the final document contains both content and the results of analysis. This functionality has been baked into R from the beginning with the Sweave function; this function took in a LaTeX document with code interspersed andproduced a final document, typically in PDF format, that “weaved” the text and the results of the code, and created a formatted final document. LaTeX was used because it (a) had very good formatting capabilities through style files, (b) allows fine control of the document, and (b) was the de facto way of writing mathematics and computer science papers.

The rmarkdown revolution

Yihui Xie created the R package knitr in 2012 as a potential replacement for Sweave. It took the best of Sweave, added features like caching code for speed, and added functionality like formatting R code, creating flexibility in handling graphics and other features. It takes a file containing text and R code and processes it by replacing code with results after running it through R.

One of the most significant changes that knitr brought was the ability to use Markdown, which is a lightweight text markup language that is easy to read and easy to write, as the source file for literate programming rather than LaTeX. This made literate programming and reporting in R far more accessible to the general R user than having to learn the rather complex LaTeX syntax, and, in my opinion, created a revolution in how the results of analyses are reported from R. This also led to the development of a particular easy syntax, rmarkdown that allowed a wide variety of documents to be created interlacing text, graphics and analytic results.

The second important aspect of using rmarkdown was the development of pandoc, a program that can take a markdown document and convert it into many different formats, including Microsoft Word, HTML, PDF (using LaTeX), and HTML slides. This made markdown a key format for generating documents for distribution and presentation. Thepandoc program is very powerful and customizable and allows the use of templates, so it allowed the automation of analytic reports from R using knitr and pandoc in combination. This pipeline was codified by the RStudio folk who developed the rmarkdown R package. The possibilities of creating analyses-driven documents using rmarkdown can be seen in the R Markdown Gallery
